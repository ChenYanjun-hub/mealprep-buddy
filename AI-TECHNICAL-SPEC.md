# 备菜助手 AI 技术说明

## 一、技术摘要

**备菜助手**是一个面向都市白领的智能菜谱推荐系统，核心能力是将用户现有食材与个人口味偏好进行语义融合，通过 LLM 生成个性化的菜谱建议与采购清单。

系统采用 **双模型热备策略**（智谱 GLM-4-flash / Claude Sonnet），配合 **Zod 结构化输出验证** 与 **三级容错机制**，实现生产级 AI 推荐稳定性。

## 三、核心技术决策

### 3.1 双模型热备策略

| 维度 | 智谱 GLM-4-flash | Claude Sonnet |
|------|------------------|---------------|
| 定位 | 主模型 | 备选模型 |
| 延迟 | ~800ms | ~1.5s |
| 成本 | ¥0.001/千token | $0.003/千token |
| 优势 | 国内访问稳定、成本低 | 输出质量高、稳定性好 |
| 场景 | 生产环境默认 | GLM 失败时降级 |

**决策理由**：智谱 API 在国内访问无延迟，成本约为 Claude 的 1/10，适合高频推荐场景。Claude 作为备选保障可用性。

### 3.2 结构化输出保障

```typescript
const RecipeSchema = z.object({
  name: z.string().min(1).max(50),
  ingredients: z.array(z.string().min(1).max(100)).min(1).max(20),
  steps: z.array(z.string().min(1).max(500)).min(1).max(20),
})

const RecommendationResponseSchema = z.object({
  recipes: z.array(RecipeSchema).min(2).max(3),
  shoppingList: z.array(z.string().min(1).max(50)).max(20),
})
```

**技术要点**：
- 后端 Zod 验证确保 AI 输出符合 Schema
- 正则提取 JSON 容错（`\`\`\`json` 包裹 / 裸 JSON）
- 验证失败自动重试，最多 3 次

### 3.3 三级容错机制

```
Level 1: AI 成功返回 + 验证通过 → 直接返回
Level 2: 验证失败 → 重试 (最多 3 次, 指数退避)
Level 3: 全部失败 → 返回 Fallback 预设菜谱
```

Fallback 策略：基于用户食材动态生成通用菜谱，保证用户体验不中断。

### 3.4 反馈闭环设计

```
用户推荐 → 查看结果 → 提交反馈 → 存储到 recipe_feedback
                                   ↓
                          未来可用于:
                          • 微调推荐策略
                          • 过滤低分菜谱
                          • A/B 测试模型效果
```

## 五、Prompt 工程

```
你是一位专业的家庭烹饪顾问，帮助打工人根据现有食材规划菜谱。

【用户信息】
- 所在城市：{city}
- 口味偏好：{tasteLabels}
- 籍贯：{hometown}

【现有食材】
{ingredients}

【任务】
请根据以上信息，推荐 2-3 道适合的菜谱：
1. 考虑用户的口味偏好和籍贯特色
2. 优先使用现有食材
3. 如需额外购买食材，请列出采购清单

【重要】必须严格按以下 JSON 格式返回：
{
  "recipes": [{ "name", "ingredients", "steps" }],
  "shoppingList": [...]
}

约束条件：
- recipes 数组必须包含 2-3 道菜
- 每道菜的 ingredients 数组至少 1 个元素
- 每道菜的 steps 数组至少 1 个元素
```

**设计要点**：
- 角色设定：专业烹饪顾问
- 上下文注入：用户偏好 + 食材清单
- 输出约束：强制 JSON 格式 + Schema 约束
- 隐式个性化：籍贯 + 口味偏好影响推荐风格

## 七、改进建议

### 短期优化
1. **流式输出**：SSE 返回，提升用户等待体验
2. **Prompt 配置化**：将 Prompt 模板抽离为配置文件
3. **缓存策略**：相同食材组合的推荐结果缓存 5 分钟

### 中期规划
1. **反馈驱动优化**：基于评分数据调整 Prompt 或微调模型
2. **多轮对话**：支持"换个做法"、"少油版"等细化需求
3. **向量检索增强**：结合菜谱知识库 RAG，提升专业度

### 长期演进
1. **个性化模型**：Fine-tuning 用户偏好的专属模型
2. **营养分析**：接入营养数据库，提供健康建议
3. **供应链对接**：采购清单直连生鲜电商平台
